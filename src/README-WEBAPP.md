# 技术群聊日报管理器 - Web App 版本

一个功能完整的网页应用，用于管理和生成技术群聊日报。

## 🎯 功能特性

### 核心功能
- ✅ **日报管理**: 创建、编辑、删除群聊日报
- ✅ **实时预览**: 即时查看日报效果
- ✅ **Markdown解析**: 智能解析群聊内容
- ✅ **本地存储**: 数据保存在浏览器本地
- ✅ **响应式设计**: 支持桌面和移动设备

### 高级功能
- ✅ **字体选择**: 7种中文字体预设
- ✅ **自定义二维码**: 上传替换默认二维码
- ✅ **多平台导出**: 支持小红书、朋友圈、即刻、Twitter格式
- ✅ **缩放预览**: 支持预览缩放调整
- ✅ **数据导入导出**: JSON格式数据备份

## 🚀 快速开始

### 1. 启动应用

```bash
# 进入项目目录
cd daily-summary-generator

# 启动本地服务器（推荐）
npx serve src -p 3000

# 或者使用 Python
cd src && python -m http.server 3000

# 或者使用 Node.js
cd src && npx http-server -p 3000
```

### 2. 访问应用

在浏览器中打开：`http://localhost:3000`

### 3. 创建第一个日报

1. 点击右上角的 **+** 按钮
2. 选择日期（默认今天）
3. 输入 Markdown 格式的群聊内容
4. 点击 **保存**

## 📝 Markdown 格式规范

### 推荐格式

```markdown
2025-10-20

内容总结：

这段聊天记录包含了两个主要议题：

1\. 技术分享：

&nbsp; - 主题: React性能优化技巧

&nbsp; - 要点: 使用React.memo, useMemo, useCallback

2\. 项目讨论：

&nbsp; - 计划: 开发新的组件库

&nbsp; - 时间线: 下周开始
```

### 格式要点

- **日期**: `YYYY-MM-DD` 格式
- **议题**: `数字\. 标题：` 格式
- **内容**: `&nbsp; - 类型: 内容` 格式
- **支持**: 自动识别和解析

## 🎨 字体选择

### 可用字体

1. **Noto Sans SC** (默认) - 现代简洁
2. **PingFang SC** - 苹果系统字体
3. **微软雅黑** - Windows 系统字体
4. **Source Han Sans CN** - 开源思源黑体
5. **站酷小薇体** - 文艺风格
6. **站酷快乐体** - 活泼风格
7. **马善政楷体** - 手写风格

### 切换方法

1. 点击左侧边栏底部的 **设置**
2. 选择喜欢的字体
3. 自动应用到所有日报

## 📱 二维码自定义

### 上传步骤

1. 进入 **设置** 页面
2. 点击 **选择图片** 按钮
3. 选择二维码图片（支持 JPG/PNG）
4. 自动应用到所有日报

### 推荐规格

- **尺寸**: 建议 200×200 像素以上
- **格式**: JPG 或 PNG
- **内容**: 群聊二维码或个人二维码

## 🖼️ PNG 导出功能

### 支持平台格式

| 平台 | 尺寸 | 比例 | 用途 |
|------|------|------|------|
| 小红书 | 1080×1440 | 3:4 | 主要引流平台 |
| 朋友圈 | 1080×1920 | 9:16 | 微信生态 |
| 即刻 | 1080×1080 | 1:1 | 技术社区 |
| Twitter | 1200×675 | 16:9 | 国际传播 |

### 导出步骤

1. 选择要导出的日报
2. 点击右上角 **导出PNG** 按钮
3. 选择目标平台格式
4. 在新窗口中手动截图保存

### 截图建议

- **Windows**: Win + Shift + S
- **Mac**: Command + Shift + 4
- **浏览器**: 使用截图扩展

## 💾 数据管理

### 本地存储

- 数据保存在浏览器的 localStorage 中
- 支持离线使用
- 无需网络连接

### 数据备份

**导出数据**:
1. 进入 **设置** 页面
2. 点击 **导出所有数据**
3. 保存 JSON 文件

**导入数据**:
1. 将 JSON 文件内容复制
2. 在浏览器控制台中执行：
   ```javascript
   app.reports = [你的JSON数据];
   app.saveReports();
   app.renderHistory();
   ```

### 清除数据

进入 **设置** 页面，点击 **清除所有数据**

## 🐛 常见问题

### Q: 预览显示空白？
A: 检查 Markdown 格式是否正确，确保日期和议题格式符合规范

### Q: 无法导出PNG？
A: 导出功能需要手动截图，新窗口打开后使用系统截图工具

### Q: 数据丢失了？
A: 检查浏览器是否清除了 localStorage，建议定期备份数据

### Q: 字体显示异常？
A: 确保网络连接正常，字体需要从 Google Fonts 加载
